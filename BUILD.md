# Build Guide

> This document explains how to build, run, and contribute to Bench.

Bench is intentionally built with a minimal technology stack.

If you're setting up the project for the first time, follow the steps in order.

---

# Supported Platforms

Bench currently targets desktop platforms.

Primary development platform:

- Windows 11

- Windows
- macOS
- Linux

# Prerequisites

Before building Bench, install:

## Git

Latest stable version.

Verify:

```bash
git --version
```

---

## Node.js

Use the current LTS release.

Verify:

```bash
node -v
npm -v
```

---

## Rust

Install using rustup.

Verify:

```bash
rustc --version
cargo --version
```

---

## Visual Studio Code

Recommended extensions:

- Rust Analyzer
- Even Better TOML
- ESLint
- Prettier
- GitLens

---

# Clone the Repository

```bash
git clone https://github.com/<username>/bench.git

cd bench
```

---

# Install Dependencies

```bash
npm install
```

---

# Start Development

```bash
npm run tauri dev
```

Bench should launch in development mode.

---

# Production Build

```bash
npm run tauri build
```

Compiled artifacts will be generated by Tauri.

# Before Opening a Pull Request

Verify:

- Documentation is updated.
- Code follows `docs/code-style.md`.
- Architecture still aligns with `docs/architecture.md`.
- New terminology has been documented.
- The project builds successfully.

---

# Troubleshooting

## Node Version Issues

Use the latest LTS version of Node.js.

---

## Rust Issues

Update Rust:

```bash
rustup update
```

---

## Dependency Problems

Clean dependencies:

```bash
rm -rf node_modules

npm install
```

Windows users can simply delete the `node_modules` folder manually if they don't have Unix tools available.

---

## Build Failures

Confirm:

- Node.js installed
- Rust installed
- Tauri prerequisites installed
- Dependencies installed

---

# Philosophy

A successful build is not enough.

A successful contribution also:

- improves readability,
- preserves simplicity,
- follows the documentation,
- and keeps Bench focused.

---

# Final Checklist

Before pushing code:

- Project builds successfully.
- Documentation matches implementation.
- No unnecessary complexity was introduced.
- Commit messages follow Conventional Commits.
- The change passes the Bench Test.

If all answers are "Yes",

you're ready to contribute.
