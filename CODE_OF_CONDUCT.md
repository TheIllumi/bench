# Code of Conduct

> Bench is built on respect, curiosity, and thoughtful engineering.

We believe great software is built by people who communicate well, challenge ideas respectfully, and assume good intent.

This Code of Conduct exists to ensure Bench remains a welcoming and productive project for everyone.

---

# Our Standards

Examples of behavior that contribute to a positive environment include:

- Being respectful and professional.
- Giving constructive feedback.
- Asking questions instead of making assumptions.
- Being open to different perspectives.
- Helping newcomers learn the project.
- Accepting criticism gracefully.
- Keeping discussions focused on improving Bench.

---

# Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment or discrimination of any kind.
- Personal attacks or insults.
- Trolling or deliberately disruptive behavior.
- Bad-faith arguments.
- Publishing private information without permission.
- Spam or self-promotion unrelated to Bench.

---

# Engineering Culture

Bench values thoughtful engineering over rapid engineering.

We encourage contributors to:

- Read the documentation before coding.
- Discuss significant changes before implementing them.
- Prefer simplicity over cleverness.
- Challenge ideas, not people.

Disagreement is expected.

Disrespect is not.

---

# AI-Assisted Contributions

AI-generated contributions are welcome.

However:

- Contributors are responsible for understanding any AI-generated code they submit.
- AI output should be reviewed before opening a pull request.
- Documentation remains the authoritative source of truth.

---

# Reporting Issues

If you experience unacceptable behavior, please open a private discussion with the maintainers or contact the repository owner.

Reports will be handled respectfully and confidentially whenever possible.

---

# Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Actions may include:

- Requesting changes.
- Removing comments.
- Closing pull requests.
- Temporarily or permanently restricting participation.

The goal is always to protect the health of the community rather than punish contributors.

---

# Final Principle

We are all here for the same reason:

To build excellent software.

Be kind.

Be curious.

Leave the project better than you found it.
