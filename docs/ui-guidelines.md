# UI Guidelines

> Bench should feel calm, intentional, and invisible.
>
> The interface exists to support thinking, not compete with it.

------------------------------------------------------------------------

# Design Philosophy

Bench is designed to remain open all day.

It should feel more like part of the operating system than a web
application.

Users should forget the UI exists and focus on their work.

------------------------------------------------------------------------

# Core Principles

## Calm

The interface should never feel busy.

Whitespace is a feature.

Avoid unnecessary borders, shadows, gradients, and visual noise.

------------------------------------------------------------------------

## Clarity

Information hierarchy should be obvious at a glance.

Important information earns visual weight.

Decoration does not.

------------------------------------------------------------------------

## Consistency

One component.

One purpose.

One appearance.

Similar interactions should always behave identically.

------------------------------------------------------------------------

## Speed

Every interaction should feel instantaneous.

Avoid animations that delay interaction.

Motion should communicate, not entertain.

------------------------------------------------------------------------

# Layout

Bench uses a three-column layout.

## Sidebar

Purpose:

Navigation between modules.

Contains only:

-   Focus
-   Projects
-   Capture
-   Lists
-   Parking Lot
-   Settings

------------------------------------------------------------------------

## Primary Panel

Displays the active module.

This is where users spend most of their time.

------------------------------------------------------------------------

## Context Panel

Displays contextual information.

Examples:

-   Notes
-   Resources
-   Project details

The panel should disappear when unnecessary.

------------------------------------------------------------------------

# Navigation

Keyboard-first.

Every primary action should have a keyboard shortcut.

Mouse support is required.

Mouse dependence is not.

------------------------------------------------------------------------

# Icons

Icons should be:

-   Minimal
-   Monochrome
-   Functional

Avoid decorative icons.

Icons should reinforce labels, never replace them.

------------------------------------------------------------------------

# Typography

Typography communicates hierarchy.

Guidelines:

-   Few font sizes
-   Consistent spacing
-   Comfortable line height
-   Strong contrast

Never use typography as decoration.

------------------------------------------------------------------------

# Color

Color communicates meaning.

Examples:

-   Success
-   Warning
-   Error
-   Focus

Avoid using color for decoration.

The default interface should remain predominantly neutral.

------------------------------------------------------------------------

# Spacing

Generous spacing reduces cognitive load.

Prefer whitespace over separators.

Avoid cramped layouts.

------------------------------------------------------------------------

# Components

Components should be:

-   Reusable
-   Predictable
-   Minimal

Each component should solve exactly one problem.

------------------------------------------------------------------------

# States

Every interactive component supports:

-   Default
-   Hover
-   Focus
-   Active
-   Disabled

These states should remain visually consistent.

------------------------------------------------------------------------

# Accessibility

Bench should be usable by everyone.

Requirements:

-   Keyboard navigation
-   Visible focus indicators
-   Adequate contrast
-   Readable typography

Accessibility is a feature.

------------------------------------------------------------------------

# Responsive Behavior

Bench is optimized for desktop.

Primary target:

-   1440p
-   Ultrawide
-   Multi-monitor setups

The layout should gracefully adapt to smaller windows.

------------------------------------------------------------------------

# Performance

The interface should:

-   Launch quickly
-   Render instantly
-   Avoid unnecessary reflows
-   Feel lightweight

------------------------------------------------------------------------

# Visual Identity

Bench should feel:

-   Professional
-   Quiet
-   Intentional
-   Durable

Never trendy.

Never flashy.

Never distracting.

------------------------------------------------------------------------

# Golden Rule

If a visual element does not improve clarity, remove it.
